# TCP/IP 网络
七层模型：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层     
四层模型：网络接口层、网络层、传输层、应用层      
- 应用层： FTP、HTTP、DNS 和 TELNET    
- 传输层：TCP 和 UDP      
- 网络层：IP ，ARP、RARP 、ICMP     
- 网络接口层：负责数据帧的发送和接收。    
域名与计算机的 IP 地址相对应，对应表存储在域名服务系统 DNS中    
可以使用命令 nslookup 或者 ping 来查看与域名相对应的 IP 地址    

端口采用 16 比特的端口号标识，一个 IP 地址的端口可以有 65536（即：2^16）个   
- SSH 22
- FTP 20 和 21
- Telnet 23
- SMTP 25
- TFTP 69
- HTTP 80
- SNMP 161
- Ping 使用ICMP，无具体端口号

# 数据链路层
数据链路层的工作就是把网络层交下来的 IP 数据报 封装为 帧（frame）发送到链路上   
其结构为 帧首部   ip报文 帧尾部
数据帧有编号保证顺序，发送后一段时间没有反馈会重发，收到数据帧后会校验（奇偶检验、CRC）  
流量控制--窗口滑动     
以太网协议-CSMA/CD 802.3   
以太网和 IEEE802.3 对数据帧长度都有限制，其最大值分别为 1500 字节和 1492 字节，链路层的这个特性称作 MTU ，即 最大传输单元
